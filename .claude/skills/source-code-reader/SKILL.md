# 源码解读博客 Skill

## 目标读者

中高级开发者 - 假设读者具备基础知识，专注于源码设计思想和实战借鉴。

## 文章结构

1. **标题**: 简洁有力，体现源码主题
2. **开场白**: 固定开场
3. **导言**: 1-2 句说明本文解读目标
4. **源码思想**: 核心设计理念
5. **源码注释**: 原代码 + 逐行/逐段注释
6. **复杂场景分析**: 边界情况和特殊处理
7. **实战借鉴**: 如何应用到实际工作
8. **总结**: 提炼关键点
9. **关注引导**: 结尾 CTA

## 写作风格

- 直接切入源码，不需要问题引入
- 先讲思想，再看代码
- 注释清晰，标注关键行
- 借鉴部分要具体，给出应用场景

## 输出规范

- 文章输出到 `./blogs/[标题].md`，标题使用源码主题命名
- 示例代码输出到 `common/engine-common/src/main/java/com/ctrip/flight/dom/engine/common/demo/[类名].java`

## 源码注释规范

```java
// 【关键】这里使用了懒加载模式
private volatile Object instance;

public Object getInstance() {
    if (instance == null) {          // 第一次检查，避免不必要的同步
        synchronized (this) {
            if (instance == null) {  // 第二次检查，防止重复创建
                instance = create();
            }
        }
    }
    return instance;
}
```
