# 技术博客编写 Skill

## 目标读者

中高级开发者 - 假设读者具备基础知识，跳过入门概念，直击核心要点。

## 文章结构

1. **标题**: 简洁有力，体现核心技术点
2. **导言**: 1-2 句说明主题和阅读收获
3. **正文**: 使用清晰的标题和小标题组织
4. **测试案例**: 完整可运行的 main 方法
5. **总结**: 提炼关键点
6. **关注引导**: 结尾 CTA

## 写作风格

- 开门见山，假设读者有经验
- 代码优先，文字为辅
- 标题层级清晰（h2/h3/h4）
- 避免基础概念解释

## 输出规范

- 文章输出到 `./blogs/[标题].md`，标题使用文章主题命名
- 示例代码输出到 `common/engine-common/src/main/java/com/ctrip/flight/dom/engine/common/demo/[类名].java`

## 测试案例规范

所有 Java 代码示例必须：
1. 包含可直接运行的 main 方法
2. 输出为独立 Java 文件到 demo 目录
3. 博客中引用代码路径

```java
package com.ctrip.flight.dom.engine.common.demo;

public class XxxDemo {
    public static void main(String[] args) {
        // 测试代码
    }
}
```
